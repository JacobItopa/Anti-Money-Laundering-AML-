# Step 2: Data Collection & Integration

## 1. Identify Data Sources
The dataset for this project was obtained from Kaggle and has been downloaded locally.
* **Dataset:** IBM Transactions for Anti Money Laundering (AML)
* **Source Link:** [Kaggle Dataset](https://www.kaggle.com/datasets/ealtman2019/ibm-transactions-for-anti-money-laundering-aml/data)

## 2. Extract and Gather
The data has been successfully extracted and moved into the project's centralized data directory. Specifically, the raw data files (`HI-Small_Trans.csv`, `HI-Small_accounts.csv`, and `HI-Small_Patterns.txt`) are securely stored in the `data/raw/` folder, ensuring the immutable original data dumps are preserved and isolated from the working code.

## 3. Data Governance & Compliance
Yes, the data is in compliance. The IBM AML dataset is a synthetic dataset generated for research and educational purposes. Because it is synthetically generated, it does not contain real Personally Identifiable Information (PII) or sensitive financial records, meaning there are no privacy or strict regulatory compliance issues (like GDPR or CCPA) regarding its usage in this project.
